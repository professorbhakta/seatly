# Competitive landscape

## vs RedBus

| | RedBus | Seatly |
|---|--------|--------|
| **Primary job** | Intercity bus **booking** across operators | GSRTC-first **awareness** + conductor ETM |
| **Passenger focus** | Search, book, pay, ticket | From→to line-up, timings, occupancy, conditional track |
| **Conductor** | Not the core product | Seat-map-first ETM, onboard + standing sales |
| **Standing visibility** | Not a first-class concept | `Seats full · N standing` before board |

**Takeaway:** RedBus is a booking marketplace. Seatly is not trying to replace that in v0.1. Seatly fills the gap between “I booked online” and “what is actually happening at the stand / on this bus right now.”

---

## vs Chalo (do not use this name)

| | Chalo | Seatly |
|---|-------|--------|
| **Primary job** | City bus live track, digital tickets, passes | GSRTC-style route awareness + conductor ETM |
| **Transit type** | Urban / city buses | State road transport (GSRTC-first) |
| **Name** | Established brand — **taken** | Seatly (working name) |

**Takeaway:** Chalo solves urban commute. Seatly targets GSRTC-style intercity / STU patterns. The name “Chalo” must not be used — it would confuse users and infringe on an existing brand space.

## Positioning summary

Seatly sits between:

- **Passenger awareness** — line-up, occupancy, track only with confirmed booking
- **Conductor tooling** — modern ETM from the seat map

It is not positioning as “the next RedBus” or “Chalo for highways” in v0.1.
